invariant.core

API facade providing invariants on Clojure data structures.

and

(and & invariants)

Ensure that all the given invariants hold.

(invariant/and
  (invariant/unique
    {:name :validator/variable-declarations-unique
     ...})
  (invariant/count
    {:name :validator/at-least-one-function-call
     ...}))

Invariant sources, targets and state will be merged internally and a combined seq of all broken invariants will be returned.

count

(count {:keys [name sources predicate]} count-predicate & args)

An invariant on the number of elements matching a given specter navigator and predicate.

For example, to ensure that there are at most 5 queries within a vector of operations.

(invariant/count
  {:name    :validator/at-most-five-queries
   :sources [:operations ALL #(= (:type %) :query)]}
  <= 5)

debug

(debug invariant data)

Run the invariant on the given data, producing a map of:

  • :result as produced by run,
  • :sources as collected from the data,
  • :targets as collected from the data,
  • :state as computed from the sources.

Should be used for debugging only, otherwise run is preferred.

invariant

(invariant {:keys [name sources targets state reduce verify], :or {targets specter/STAY, state (), reduce conj}})

Generate an invariant describing the relationship between two subsets of a given piece of data.

  • :name: the name of the invariant (needs to be a namespaced keyword),
  • :sources: specter paths to elements the invariant state is built from,
  • :targets: specter paths to elements that are verified using the invariant,
  • :state: the initial invariant state,
  • :reduce: a reducer function to be applied to :state and source items,
  • :verify: a predicate applied to verification state and target items.

Internally, the incoming data is traversed using :sources paths and each element passed to :reduce. Afterwards, the final state is used to verify all :targets paths using :verify.

For example, to create an invariant between variable declarations and usages:

(invariant/invariant
  {:name    :validator/variables-have-been-declared-before-usage
   :sources [:declarations ALL :name]
   :targets [:body (walker :variable) :variable (must :name)]
   :state   #{}
   :reduce  conj
   :verify  contains?})

This will collect all variable names and store them in a set, before verifying that the name of each used variable appears within said set.

unique

(unique {:keys [name sources unique-by], :or {unique-by identity}})

A uniqueness invariant on all elements matching the specter navigator given in :sources. :unique-by can be used to generate the value that has to be unique for each element (default: identity).

For example, to verify that each variable is only declared once:

(invariant/unique
  {:name   :validator/declarations-unique
   :sources [:declarations ALL :name]})
